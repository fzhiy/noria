# NORIA Schema

> Full schema definition for the wiki. Loaded by skills (kb-compile, kb-lint, etc.) on demand.
> CLAUDE.md references this file but does NOT inline its content.

## Provenance Levels

- **`source-derived`**: Directly extracted from `raw/`. High trust.
- **`llm-derived`**: Synthesized across ≥2 sources. Medium trust.
- **`user-verified`**: Manually confirmed by user. Highest trust.
- **`social-lead`**: Social media / grey literature. Low trust. Cannot cite in synthesis.
- **`query-derived`**: Generated by `/kb-ask`. Lowest trust. Lives in `outputs/` only.

### Provenance Rules

1. `query-derived` content NEVER enters `wiki/` or `index.md`.
2. `social-lead` NEVER counts toward synthesis citation requirements (≥2 `source-derived` needed).
3. `social-lead` pages MUST NOT use `venue_tier: top-conf` or `venue_tier: top-journal`.
4. Social claims corroborated by peer-reviewed source → upgrade to `source-derived`.
5. Preserve existing `user-verified` content. Only add, don't overwrite.
6. New source contradicts existing content → flag contradiction explicitly.

## Wiki Page Format

All pages use this frontmatter template:

```yaml
---
title: "Page Title"
type: source | concept | synthesis
provenance: source-derived | llm-derived | user-verified | social-lead
sources: [citekey1, citekey2]
tags: [keyword1, keyword2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

### Provenance assignment during kb-compile

| Raw source directory | Default provenance | Notes |
|---|---|---|
| `raw/zotero/`, `raw/arxiv/`, `raw/semantic-scholar/` | `source-derived` | Academic papers |
| `raw/twitter/`, `raw/wechat/`, `raw/github/` | **`social-lead`** | Social/grey literature |
| `raw/notes/` | `llm-derived` | Analysis notes |

**Exception**: A GitHub repo that IS the official artifact of a compiled paper may use `source-derived` with the paper's citekey in `sources:`.

### Bibliographic metadata (required for source pages)

```yaml
authors: [First Author, Second Author]
year: 2025
venue: "ICLR 2025"
venue_tier: top-conf | top-journal | workshop | arxiv-preprint | social
venue_verified: true | false
venue_verification_source: "s2"  # s2 | openreview | dblp | manual | auto-unverified
doi: "10.xxxx/yyyy"
citation_count: 148
institution: [University, Lab]
github_url: "https://github.com/org/repo"
code_available: true | false | unknown
domain: agent | peft | drift | benchmark | foundational | other
```

**domain classification**: `agent` (web/GUI agents), `peft` (fine-tuning/merging), `drift` (UI/workflow drift), `benchmark` (evaluation), `foundational` (continual learning), `other`

### Domain-specific comparison fields (optional, added by kb-deepen)

These structured fields enable Dataview/Bases filtering and cross-paper comparison. Only populate fields relevant to the paper — omit inapplicable ones.

```yaml
drift_type: ui-drift | workflow-drift | concept-drift | data-drift | none
adaptation_mode: rl | sft | gradient-free | prompt-only | hybrid | none
benchmark_type: interactive-env | static-dataset | live-web | simulation | none
recovery_metric: success-rate | task-completion | step-accuracy | reward | other
safety_scope: adversarial | alignment | robustness | privacy | none
memory_form: parametric | episodic | external-kb | hybrid | none
```

- **drift_type**: What kind of environment change the paper addresses
- **adaptation_mode**: How the agent adapts (RL training, supervised fine-tuning, gradient-free, prompt-only)
- **benchmark_type**: Evaluation environment type
- **recovery_metric**: Primary metric for measuring recovery/adaptation success
- **safety_scope**: Which safety dimension the paper targets
- **memory_form**: How the agent stores and retrieves experience

These fields are NOT lint-required. Lint ignores unknown frontmatter keys.

**venue_tier**: `top-conf` (CCF-A: NeurIPS, ICML, ICLR, AAAI, ACL, CVPR, WWW, KDD...), `top-journal` (TPAMI, JMLR, TMLR...), `workshop`, `arxiv-preprint`, `social`

**venue_verified rules**: When `venue_tier` is `top-conf`/`top-journal`, MUST verify on official sources (OpenReview, ACL Anthology, PMLR, IEEE Xplore, DOI). Never trust PDF self-declared venue. Run `npx tsx tools/venue-verify.ts <citekey>`.

### Entity pages (optional, for high-frequency entities)

For research labs, researchers, or projects that appear across many source pages, create dedicated entity pages in `wiki/entities/`:

```yaml
---
title: "Lab/Person/Project Name"
type: entity
entity_type: lab | researcher | project | dataset | benchmark-suite
provenance: llm-derived
sources: [citekeys of papers from this entity]
tags: [entity:lab-name, relevant-domain-tags]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

Entity pages are lightweight profile pages linking to the entity's source contributions. They provide navigation hubs in the Obsidian graph — connecting papers from the same lab/author.

**When to create**: entity appears in ≥3 source pages. Do NOT create for every author — only for high-frequency entities that serve as navigation hubs.

### Entity metadata (tag-based, for lower-frequency entities)

Use tags: `author:hinton`, `method:transformer`, `dataset:imagenet`, `venue:neurips`, `benchmark:glue`.

### Citation format

```
The attention mechanism scales quadratically [source: vaswani2017, abstract].
```

**Locator rules**: `abstract` (Zotero imports), `sec.X.Y` (when section content available), `title`, `webpage`, `tweet`, `readme`, `release-notes`. Never fabricate section numbers.

### Cross-references

Obsidian-compatible wikilinks: `[[concept-slug|Display Name]]`, `[[source-slug]]`.

## Conflict Handling

- New source contradicts existing → add `## Contradictions` section with both claims cited.
- Duplicate concept pages (fuzzy match) → flag for human review, do NOT auto-merge.
- Retracted source → mark `status: superseded`, move to `wiki/archive/`, link replacement.
- Already-compiled file → skip (manifest check). Use `--force` to recompile.
