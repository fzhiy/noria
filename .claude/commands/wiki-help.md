## Section 1: Quick Summary (always show for `--quick` or full)

Print this:

> **NORIA** is a provenance-first research knowledge base maintained by Claude Code with Obsidian as the human frontend.
> Papers flow from Zotero/arXiv/Semantic Scholar → `raw/` → `/kb-compile` → `wiki/` (sources + concepts + synthesis).
> Every claim is citation-traced, every page has provenance, and `/kb-lint` gates all synthesis.
> The wiki serves as a cross-project knowledge accumulation layer accessible via QMD MCP, JSONL export, or topic bundles.

## Section 3: Provenance Model (show for `--provenance` or full)

Print this:

| Level | Trust | Rule | Example |
|---|---|---|---|
| `source-derived` | High | Directly from raw/ file | Paper summary, key claims |
| `llm-derived` | Medium | Synthesized across ≥2 sources | Concept pages |
| `user-verified` | Highest | Manually confirmed by user | Human-reviewed sections |
| `query-derived` | Low | Generated by /kb-ask | **NEVER enters wiki/** |

**Core constraints:**
- Every non-trivial claim must cite `[source: citekey, location]`
- Citation locators: `abstract` (most common), `sec.X.Y` (only if PDF was read), `title`, `webpage`
- Never fabricate section numbers for abstract-only sources
- Contradictions are flagged explicitly, never silently overwritten

## Section 5: Current Status (show for `--status` or full)

**Dynamically generate** by running these checks:

1. Count pages:
   ```bash
   echo "Sources: $(ls wiki/sources/*.md 2>/dev/null | wc -l)"
   echo "Concepts: $(ls wiki/concepts/*.md 2>/dev/null | wc -l)"
   echo "Synthesis: $(ls wiki/synthesis/*.md 2>/dev/null | grep -v gitkeep | wc -l)"
   ```

2. Run lint (quick):
   ```bash
   npx tsx tools/kb-lint.ts --json 2>/dev/null | python3 -c "import sys,json; d=json.load(sys.stdin); print(f'Lint: {d[\"summary\"][\"passed\"]}/{d[\"summary\"][\"passed\"]+d[\"summary\"][\"failed\"]} PASS')"
   ```

3. Latest commit:
   ```bash
   git log --oneline -1
   ```

4. Manifest state:
   ```bash
   python3 -c "import json; m=json.load(open('.kb/manifest.json')); c=sum(1 for v in m['files'].values() if v.get('status')=='compiled'); p=sum(1 for v in m['files'].values() if v.get('status')=='pruned'); print(f'Manifest: {c} compiled, {p} pruned')"
   ```

5. Recommend next action based on status:
   - If synthesis/ is empty → "Ready for /kb-reflect (lint passes, recommend topic: Continual Adaptation Methods for Web Agents)"
   - If lint fails → "Run /kb-lint --fix first"
   - If new raw files exist → "Run /kb-compile to process new sources"
   - Otherwise → "Wiki is healthy. Consider /kb-sync to expand sources."

## Section 6: Typical Workflows (show for `--workflow` or full)

Print these flows:

**First-time setup:**
```
/kb-sync                    # Sync papers from Zotero
/kb-compile                 # Generate source + concept pages
/kb-lint                    # Verify structure (must 7/7 PASS)
/kb-reflect                 # Generate synthesis articles
```

**Add new papers:**
```
/kb-sync --collection "Agent"         # Sync specific collection
/kb-sync arxiv "web agent" --limit 5  # Or search arXiv
/kb-sync s2 "web agent" --year 2024-  # Or search Semantic Scholar
/kb-compile                           # Compile new sources
/kb-lint                              # Verify
```

**Deepen existing papers:**
```
/kb-deepen cai2026-building-self-evolvi   # Read PDF, enrich source page
/kb-lint                                   # Verify
```

**Query knowledge:**
```
/kb-ask "What methods exist for continual agent learning?"
# Or for external projects:
npx tsx tools/kb-topic-bundle.ts "web agent" --format json
```

**Export for external use:**
```
/kb-output --format jsonl              # LLM context stuffing
/kb-output --format marp --slug web-agent  # Presentation slides
```
